# 설정 취향

## 목적

dotfiles 설정과 워크플로 결정 시 따라야 할 개인 원칙. 이 원칙과 어긋나는 방향으로 작업하기 전에 반드시 사용자에게 확인받는다.

## 원칙

### 1. 미니멀리즘 (Minimalism)

프롬프트나 설정에 부수적 컨텍스트(AWS 프로파일, 시간, k8s 컨텍스트 등)를 자동으로 추가하지 않는다. 꼭 필요한 것만 남긴다.

### 2. Silent enhancement

새로운 alias / 슬래시 명령 / 키바인딩을 추가해서 사용자가 외워야 하는 방향보다, **기존 명령이 조용히 더 강해지는** 방향을 선호한다.

### 3. Local vs tracked 분리

- **tracked** (`zshrc`, `gitconfig` 등): 모든 장비에 보편적인 설정
- **`.local`** (`zshrc.local`, `gitconfig.local` 등): 머신/계정 고유값 (이메일, 절대 경로, 회사 프로파일 등)

보편적인 개선은 tracked, 머신 고유값은 `.local`에만 둔다.

### 4. 프로젝트 비의존

dotfiles는 특정 작업 프로젝트에 디펜던시를 두지 않는다. 모든 장비에서 그 자체로 동작해야 한다.

디펜던시뿐 아니라 **회사·프로젝트 이름 자체도 tracked 파일(코드, 설정, 문서, 커밋 메시지)에 노출하지 않는다.**

### 5. 암묵적 스타일 default 금지

명시되지 않은 형식 결정(markdown 줄바꿈, 들여쓰기, 따옴표, 주석 스타일 등)을 할 때:

- 프로젝트 내 기존 파일에 패턴이 있으면 그것을 따른다
- 없으면 학습된 default를 묵묵히 적용하지 않고 사용자에게 묻는다
- 한 번 정해지면 같은 프로젝트 안에서 일관 적용한다

이 dotfiles의 markdown은 **soft wrap** (한 문단 = 한 줄)을 사용한다.

## 충돌 시 처리 방식

위 원칙과 어긋나는 요청을 받았을 때:

1. **묵묵히 따르지 않는다** — 그대로 적용하면 나중에 후회할 가능성이 높음
2. **충돌 사실을 짚는다** — 어떤 원칙과 어떻게 어긋나는지 한 줄로 설명
3. **확인을 받는다** — "그럼에도 진행할지" 또는 "다른 접근이 있는지" 물어본다
